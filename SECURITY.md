## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly by emailing **security@pilos.net**.

Please do **not** open a public GitHub issue for security vulnerabilities.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Potential impact assessment
- Any suggested fix (optional)

### Response Timeline

- **Acknowledgment:** Within 48 hours of your report
- **Initial assessment:** Within 5 business days
- **Fix for critical issues:** Within 7 days
- **Fix for high-severity issues:** Within 14 days
- **Fix for medium/low issues:** Within 30 days

### Severity Classification

| Severity | Description | Examples |
| -------- | ----------- | -------- |
| **Critical** | Actively exploitable, affects all users, no workaround | Remote code execution, authentication bypass |
| **High** | Significant risk, limited exploitation | Privilege escalation, data exposure |
| **Medium** | Moderate risk, requires specific conditions | Cross-site scripting, information leakage |
| **Low** | Minimal risk, unlikely exploitation | Minor information disclosure, best practice violations |

### Disclosure Policy

We follow a coordinated disclosure process:

1. Reporter submits vulnerability to security@pilos.net
2. We acknowledge and assess the report
3. We develop and test a fix
4. We release the fix and publish a security advisory
5. Reporter is credited (unless they prefer anonymity)

We ask reporters to allow up to 90 days before public disclosure to give us time to develop and distribute a fix.
